
# ElementalTetris

## 概要
本作 は、テトリスに効果演算を導入しています。  
**元素反応付与テトリスゲーム**です。

通常のテトリスと同様にラインを消去しますが、  
ライン内に含まれる元素の組み合わせによって  
**特殊な元素反応（追加効果）が発生**します。

操作技術だけでなく、  
「どの元素を組み合わせて消すか」という  
**戦略性**を重視した設計になっています。

---

## ゲームの特徴

- 各ブロックは元素属性を持つ
- ライン消去時に元素反応が発生
- 元素反応によってスコア・盤面・挙動が変化
- ScriptableObject を用いた **データ駆動設計**

---

## 元素反応一覧

| 組み合わせ | 反応名 | 効果 |
|----------|------|------|
| 炎 + 水 | 蒸発 | スコア2倍 |
| 炎 + 氷 | 溶解 | 周囲1マス消去 |
| 雷 + 水 | 感電 | 次のブロックが自動回転 |
| 氷 + 水 | 凍結 | 一定時間落下速度低下 |
| 炎 + 草 | 燃焼 | 数秒間自動で列消去 |
| 風 + 他元素 | 拡散 | 同元素を周囲にばらまく |
| 岩 + 任意 | 結晶化 | ランダムな1列を消去 |

**反応内容はすべてデータとして定義されています。

## 設計方針

本作では **データ駆動設計** を採用しています。

### 方針
- ゲームルールや数値は **コードに直接書かない**
- ScriptableObject によって **データとして定義**
- コードは「仕組み」の実装に専念

### メリット
- 元素・反応・効果の追加が容易
- バランス調整がデータ編集のみで可能
- 拡張性・保守性が高い

## アーキテクチャ構成

Assets/
├─ Scripts/
│  ├─ Domain/            # ゲームの概念・ルール（純粋ロジック）
│  ├─ Data/              # ScriptableObject 用データ定義
│  ├─ Managers/          # ゲーム進行・全体制御
│  └─ Tetris/            # テトリス固有ロジック
│
├─ ScriptableObjects/
│  ├─ Elements/          # 元素データ
│  ├─ Reactions/         # 元素反応データ
│  └─ Effects/           # 効果データ
│
├─ Scenes/               # シーン定義
└─ Prefabs/              # プレハブ

---

## データ駆動の仕組み(Data配下)

### ElementData
- 元素名
- 表示色などの見た目情報

### ReactionData
- 元素の組み合わせ
- 発生する効果の定義

### EffectData
- ダメージ・回復・バフなどの「効果」を定義
・スコア×2
・条件付き効果等
  　→実際の処理はScoreManagerで実行


**今後の新しい元素・組み合わせ、バフ等の効果処理は Data配下追加する
